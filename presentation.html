API Requirements
TECHNICAL REQUIREMENTS

What We Need from QuantBD

Detailed API & data requirements for DSE matching engine integration.

1 Real-Time Market Data API

  • Live Price Feed — LTP, bid/ask, volume, change% for all DSE instruments via WebSocket or SSE
  • Market Depth (Level 2) — Top 5-10 bid/ask levels with quantities for order book visualization
  • Index Data — DSEX, DSES, DS30 real-time values with intraday OHLC
  • Trade Ticker — Last N trades stream with price, volume, time, buyer/seller type
  • Market Status — Pre-open, continuous, closing, post-close session indicators

Format: JSON via WebSocket (preferred) or REST polling ≤1s latency · Auth: API key + HMAC signature

2 Historical Data API

  • OHLCV Bars — Daily/weekly/monthly candles for all instruments (min 5 years)
  • Intraday Bars — 1min, 5min, 15min, 1hr candles for charting
  • Corporate Actions — Dividends, rights, bonuses, splits, AGM dates
  • IPO History — Subscription dates, prices, lot sizes, oversubscription ratios
  • Market Indices History — Daily DSEX/DSES/DS30 with turnover & volume

Format: REST API with pagination · CSV/JSON bulk export for initial backfill · Delta sync for daily updates

3 Fundamental Data API

  • Financial Statements — Income statement, balance sheet, cash flow (quarterly + annual)
  • Key Ratios — EPS, P/E, NAV, dividend yield, ROE, debt-to-equity
  • Company Profiles — Sector, category, listing date, authorized/paid-up capital
  • Shareholding Pattern — Sponsor, institutional, foreign, public holdings %
  • News & Announcements — DSE/BSEC circulars, company price-sensitive info (PSI)

Format: REST API · Updated within 24hrs of DSE/BSEC publication · Webhook for PSI alerts

4 Order & Execution API (Phase 2)

  • Order Placement — Market/limit orders via FIX protocol or REST gateway to DSE matching engine
  • Order Status — Real-time execution reports, partial fills, rejections
  • Portfolio Sync — CDBL BO account integration for holdings reconciliation
  • Settlement Info — T+2 settlement tracking, margin requirements
  • Broker Routing — Multi-broker order routing through QuantBD gateway

Phase 2 (Month 7+) · Requires BSEC-licensed broker partnership · FIX 4.4 or REST gateway

Legal & Compliance
COMPLIANCE

Regulatory Roadmap & Legal Considerations

!

Key Legal Requirements

  • BSEC Registration — Required for investment advisory platforms in Bangladesh
  • Data Usage License — DSE/CSE data redistribution requires formal agreement
  • KYC/AML Compliance — Bangladesh Bank guidelines for user verification
  • Consumer Protection — Clear disclaimers: education ≠ financial advice
  • Data Privacy — User data handling per Digital Security Act 2018
  • Company Registration — RJSC registration as fintech company
✓

Our Mitigation Strategy

  • Education-first positioning avoids brokerage licensing initially
  • Demo trading with virtual coins = no real financial risk
  • Partnership with licensed broker for eventual real trading
  • Legal counsel engaged for BSEC advisory registration
  • KYC system already built and operational
  • All content carries "not financial advice" disclaimers
  • QuantBD data partnership solves DSE licensing cleanly
